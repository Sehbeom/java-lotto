### <전 주차 개선점 적용>

- TDD 방식 유지하기
- 가독성을 위한 메소드 네이밍 방식 적용해보기
    - 로직을 직접적으로 포함하고 있지 않은 상위 메소드의 경우, 기능 위주로 네이밍
    - 로직을 직접적으로 포함하고 있는 하위 메소드의 경우, 로직 위주로 네이밍
- 커밋 메세지의 scope를 패키지 단위로 채우기
    - controller, model, view, exception, constants 등
- 최대한 작은 단위로 테스트 코드 작성하기



### <새로운 시도>
